# Подключение формы записи к Google-таблице

Форма на странице `index.html` отправляет имя и комментарий в Google-таблицу
через веб-приложение Google Apps Script. Сейчас запись **не доходит**: эндпоинт
в `index.html` отвечает `HTTP 401` — значит развёртывание либо недоступно «всем»,
либо URL устарел. Ниже — как починить с нуля.

## Диагноз

Проверить эндпоинт можно так (PowerShell):

```powershell
curl.exe -L "https://script.google.com/macros/s/.../exec"
```

- Если открывается JSON `{"ok":true,"status":"alive"}` — деплой живой и публичный.
- Если `401` / страница «Не удаётся открыть файл» — доступ закрыт или URL мёртвый.

> Важно: из-за `mode:'no-cors'` в браузере страница **всегда** показывает «✓ Готово»,
> даже когда сервер ответил ошибкой. Поэтому судить об успехе нужно по таблице,
> а не по сообщению на форме.

## Шаги настройки

1. Открыть нужную Google-таблицу → **Расширения → Apps Script**.
2. Вставить код из файла [`apps-script.gs`](apps-script.gs), сохранить.
3. **Развёртывание → Новое развёртывание**, тип — **Веб-приложение**:
   - **Запуск от имени:** Я (Me)
   - **У кого есть доступ:** **ВСЕ / Anyone** ← ключевой пункт, без него `401`
4. Подтвердить доступы (Google спросит разрешение на запись в таблицу).
5. Скопировать URL развёртывания вида `https://script.google.com/macros/s/.../exec`.
6. Вставить его в `index.html` в строку:
   ```js
   const ENDPOINT = "СЮДА_НОВЫЙ_URL";
   ```
7. Проверить: открыть страницу, нажать «Записаться», заполнить имя → в таблице
   должна появиться строка с датой, именем и комментарием.

## После правок кода скрипта

Если меняете `apps-script.gs`, не делайте «Новое развёртывание» (оно даёт новый URL).
Идите в **Управление развёртываниями → ✏ редактировать → Версия: Новая версия**.
Тогда URL `/exec` останется прежним и `ENDPOINT` менять не нужно.

## Частые причины «строка не добавляется»

| Симптом | Причина | Что делать |
|---|---|---|
| `401` на `/exec` | доступ не «Все» или URL устарел | передеплой с доступом Anyone, обновить ENDPOINT |
| `200`, но строки нет | пишет не в тот лист | проверить `SHEET_NAME` в `apps-script.gs` |
| Работало, потом перестало | переразвернули как «новое» → новый URL | вставить актуальный `/exec` в ENDPOINT |
